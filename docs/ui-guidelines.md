# TidyNest UI Guidelines

## UX Principles

- Optimize for quick daily interactions
- Keep common actions visible and low-friction
- Favor progressive disclosure for advanced options

## Visual Direction

- Use a calm, clean visual style suitable for productivity
- Maintain clear visual hierarchy (headings, spacing, emphasis)
- Avoid noisy decorative elements that reduce task focus

## Responsive Behavior

- Mobile-first layout decisions for day-to-day usage
- Ensure desktop view adds density without clutter
- Keep touch targets accessible on small screens

## Component Patterns

- Build reusable primitives for inputs, buttons, cards, and lists
- Keep page components compositional, not monolithic
- Prefer predictable state transitions and loading states

## Content and Tone

- Use concise, action-oriented labels
- Provide helpful empty-state guidance
- Error messages should explain next action

## Accessibility

- Ensure keyboard navigation works for core flows
- Maintain sufficient color contrast
- Use semantic HTML and accessible labels

## Interaction Quality

- Use subtle transitions to reinforce state changes
- Avoid excessive animation; prioritize clarity and speed
- Show optimistic feedback only when safe to do so
